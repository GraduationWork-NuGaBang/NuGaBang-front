## var 키워드

1. 중복 생성이 가능
    1. 중복 선언을 하면 초기화문의 유무에 따라 다르게 동작하는데 초기화문이 있으면 var 키워드가 없는것처럼 동작하고 없으면 변수 선언이 무시가되는데 에러는 발생하지 않음
2. 함수 레벨 스코프
    1. var키워드는 오로지 코드 블록만을 지역 스코프로 인정 코드 블록 내에서 생성해도 모두 전역 변수
3. 변수 호스팅
    1. var키워드로 선언하면 선언문 이전에도 참조 가능

### let키워드

1. 변수 중복 선언 금지
2. 블록 레벨 스코프
    1. 모든 코드 블록을 지역 스코프 인정
3. 변수 호스팅
    1. let키워드로 선언한 변수는 선언단계와 초기화 단계가 분리되어 진행된다. 또한 선언문 이전에 참조하면 에러 let 키워드로 생성하면 시작부터 초기화단계 시작 지점까지 변수를 참조할 수 없음

### const키워드

대부분이 let키워드와 비슷하다.

하지만 let과 다른점은...?

1. const 키워드로 선언한 변수는 반드시 선언과 동시에 초기화해야한다.
2. 다른 키워드와 다르게 재할당이 금지된다.
3. 상수는 재할당이 금지된 변수인데 const로 선언한 변수에 원시값을 할당하면 변수 값을 변경할 수 없음.
4. 변수에 객체를 할당한 경우에만 값을 변경 가능

### 결론적으로 세개 중에 사용하기 좋은 키워드는?

변수선언에 기본적으로는 const를 사용하고 재할당이 필요하면 let을 사용하는데 한정해서 사용하는 것이 좋다. 그래도const를 사용하면 의도치 않은 재할당을 방지하여 안전
